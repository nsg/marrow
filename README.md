## About

Marrow is a model-agnostic agent framework that dynamically generates the tools it needs, self-heals when things break, and keeps everything observable. No hardcoded integrations. No black boxes.

It monitors your inputs (calendar, communications, tasks), understands intent through conversation, assembles lean per-task context, and acts — generating sandboxed Lua connectors on the fly for any service it needs to reach.

## Features

- **Zero hardcoded integrations** — all connectors are Lua scripts generated by the model at runtime, sandboxed and validated
- **Smart triage** — a fast model call decides if external data is needed before running the tool pipeline, skipping unnecessary work for conversational follow-ups
- **Self-healing janitor** — background process reviews generated tools, catches issues (hardcoded values, missing error handling, description mismatches), regenerates fixes, and escalates when it can't repair
- **Working memory** — learns facts from interactions, recalls relevant context per task, avoids duplicates
- **Conversation history** — interactive sessions maintain message history with automatic summarization
- **Model routing** — role-based config maps tasks to models (fast, code, analytical), all via TOML
- **Full transparency** — structured JSONL event log with progressive detail (`--verbose` for full stream)
- **Parameterized tools** — tool selection extracts parameters from task descriptions (e.g., location, timezone) and passes them as globals to Lua scripts

## Architecture

```
User Input
  → Triage: needs external data? (yes/no)
  → If yes: Tool Selection + Parameter Extraction
    → If no match: Code Generation → Test Run → Save to Toolbox
  → Memory Retrieval (relevant facts)
  → Context Assembly (Lua sandbox + memories)
  → Model Router → Ollama Backend (local/cloud)
  → Response
  → Memory Writer (auto-save new facts)
  → Janitor (async background validation)
```

| Component | Description |
|---|---|
| **Orchestration** | Task lifecycle, state transitions, concurrent execution |
| **Triage** | Fast model decides if tools are needed before selection |
| **Tool Selection** | Model picks tools + extracts parameters from task description |
| **Code Generation** | Model writes Lua scripts when no existing tool matches |
| **Lua Sandbox** | Secure runtime — all I/O through controlled Rust host functions (`http_get`, `http_post`, `json_parse`, `json_encode`, `log`) |
| **Toolbox** | Directory of validated Lua scripts with TOML metadata |
| **Janitor** | Async reviewer — validates code matches description, regenerates on failure, escalates after 3 attempts |
| **Working Memory** | JSON files (one fact per file), model-selected per task, auto-saved post-interaction |
| **Session History** | Per-session message history with automatic summarization at threshold |
| **Event Log** | JSONL append-only log of all pipeline events |
| **Model Router** | Role-based dispatch (fast/code/default) to Ollama backends |

## Quick Start

```sh
# Build
cargo build

# Copy and edit config
cp config.example.toml config.toml
# Set your model, provider, and API key in config.toml

# Interactive mode
cargo run

# Single prompt
cargo run -- -p "What is the weather in Tokyo?"

# Verbose mode (full event stream)
cargo run -- --verbose -p "What time is it?"
```

## Configuration

Model routing is configured in `config.toml`:

```toml
[roles.default]
provider = "ollama"
model = "mistral"
api_base = "https://ollama.com"
api_key = "your-key"

[roles.fast]
provider = "ollama"
model = "mistral"
api_base = "https://ollama.com"
api_key = "your-key"

[roles.code]
provider = "ollama"
model = "deepseek-coder"
api_base = "https://ollama.com"
api_key = "your-key"
```

Omit `api_base` to use local Ollama at `localhost:11434`. Omit `api_key` for local instances.

## CLI Options

| Flag | Default | Description |
|---|---|---|
| `-c, --config` | `config.toml` | Path to config file |
| `-r, --role` | `default` | Model role for task execution |
| `-p, --prompt` | — | Single prompt (non-interactive) |
| `-t, --toolbox` | `toolbox` | Path to toolbox directory |
| `-m, --memory` | `memory` | Path to memory directory |
| `-v, --verbose` | off | Show full event stream |
| `--log` | `events.jsonl` | Path to event log file |
